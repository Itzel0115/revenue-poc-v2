# 研究進度報告

## 1. 研究目的

本研究以「營收去識別化先驗測試」為主題，目的是在不直接暴露敏感營運資訊的前提下，建立一套可支援營收與庫存分析的資料處理與問答系統。研究核心在於驗證：去識別化後的資料，是否仍可保有足夠的分析價值，並進一步支援管理判讀、異常偵測與互動式查詢。

本研究的具體目標如下：

- 建立營收、庫存與 mapping 資料的標準化分析流程。
- 將去識別化規則納入資料轉換與分析過程。
- 驗證去識別化資料在趨勢分析、群組分析、異常分析與關聯分析上的可行性。
- 在既有分析流程上，進一步加入自然語言問答與 multi-agent 協作機制。
- 建立可展示、可擴充的原型系統，作為後續平台化發展基礎。

## 2. 研究方法

本研究採用「資料分析流程建構」與「AI 問答能力擴充」並行的方式進行，整體方法如下。

### 2.1 資料蒐整與前處理

研究以三份 Excel 檔案作為主要輸入來源：

- `inventory.xlsx`
- `revenue.xlsx`
- `mapping.xlsx`

首先進行欄位檢核、日期格式統一、數值格式整理與缺漏檢查，確保後續分析建立在可用且一致的資料基礎上。

### 2.2 Mapping 與去識別化規則解析

透過 mapping 檔案，將事業群、HQBU、平台與去識別化處理方式轉為結構化規則，用以支持：

- 事業群分類
- HQBU 對應
- 平台對應
- 金額 / QTY / 營收欄位處理邏輯
- 跨表橋接候選關係建立

此步驟的重點在於，讓資料在去識別化後仍保有分析上的對應基礎。

### 2.3 分析模型與規則設計

在分析方法上，目前以規則式與彙總式分析為主，包含：

- 月趨勢分析
- 事業群排名分析
- 平台月度整合分析
- 營收 / 庫存代理比值分析
- 異常偵測
- 相關分析

異常偵測主要根據月增率與比值門檻進行判讀；關聯分析則先以 correlation 作為主要方法，作為 transaction-level association rules 未完成前的替代方案。

### 2.4 AI 與 Multi-Agent 方法

在既有分析管線之上，增加 multi-agent 問答層，將自然語言問題拆解為多個分析任務，再交由不同 Agent 分工處理：

- Main Agent：問題理解、路由、任務拆解與結果整合
- Sales Agent：營收分析
- Inventory Agent：庫存分析
- Financial Agent：代理財務指標與異常判讀
- Association Agent：相關分析與 fallback 解釋

此方法的目的是讓系統不只輸出固定報表，也能支援互動式問題查詢與動態說明。

## 3. 目前研究結果

截至目前為止，本研究已完成可運作的原型系統，主要成果如下。

### 3.1 已完成之系統能力

- 已完成 Excel 資料讀取、欄位檢核與前處理流程。
- 已完成 mapping 結構化解析與部分 HQBU / 平台橋接機制。
- 已完成營收、庫存金額、庫存 QTY 的月趨勢分析。
- 已完成事業群層級的聚合與排名分析。
- 已完成部分平台層級的營收 / 庫存代理比值分析。
- 已完成異常偵測邏輯。
- 已完成相關分析邏輯。
- 已完成 Markdown 報告、Excel 彙整檔與圖表輸出。
- 已完成 CLI 問答模式。
- 已完成 multi-agent 問答原型。
- 已完成可展示的 Web Demo 介面。

### 3.2 目前已驗證的研究價值

從目前成果可初步驗證以下幾點：

- 去識別化資料在月份、平台與事業群層級，仍可支援管理分析。
- 去識別化後仍可進行趨勢追蹤、排名判讀與異常發現。
- 透過 mapping 與橋接規則，可建立有限但可用的跨表分析基礎。
- 在既有分析結果之上加入 Agent 與 Tool Layer，可以有效支援自然語言問答。
- 即使 LLM 不可用，透過 fallback routing 與既有分析結果，系統仍具基本可用性。

### 3.3 目前研究限制

雖然原型已具備初步成果，但目前仍有明確限制：

- 尚未支援 transaction-level association rules。
- 尚未支援完整財務報表分析。
- 尚未具備商品 / SKU 層級分析能力。
- HQBU 與平台的橋接仍有不穩定情況，跨表分析能力受到資料結構限制。
- 執行環境與測試機制尚未完全標準化，尚屬原型驗證階段。

## 4. 未來研究結果與預期方向

後續研究將朝向「分析深化」、「AI 能力提升」與「系統平台化」三個方向發展。

### 4.1 分析深化

預期未來可達成的成果包括：

- 導入更多資料來源，例如採購、出貨、成本、毛利與費用。
- 擴大分析粒度至商品、SKU、客戶或通路層級。
- 提升跨表對應的完整性與穩定性。
- 加入預測模型與預警機制，例如營收預測與庫存風險提示。

若上述資料條件補足，系統將可從「歷史分析工具」進一步發展為「營運決策支援工具」。

### 4.2 AI 問答與 Agent 能力提升

預期後續可進一步完成：

- 強化 Main Agent 的問題理解與任務拆解準確度。
- 增加新的 domain agents，例如 forecast agent、risk agent、executive summary agent。
- 將 correlation fallback 逐步升級為更完整的關聯規則分析。
- 強化回答品質評估、模板管理與模型切換策略。

此方向的成果將使系統從單純問答介面，提升為可主動提供判讀建議與管理摘要的智能分析助手。

### 4.3 系統化與平台化

預期未來可逐步完成：

- 正式化前端介面與使用流程
- 權限控管與操作紀錄
- 排程執行與週期性報告產出
- 更完整的 logging、監控與稽核能力
- 自動化測試與部署流程

若能完成上述項目，系統將可由 PoC 演進為正式的內部分析平台。

## 5. 結論

本研究目前已完成一套具備資料清洗、去識別化規則解析、分析輸出與 multi-agent 問答功能的原型系統，初步證明去識別化後的營收與庫存資料，仍可保有實際分析價值，並可透過 AI 問答方式提升使用彈性與解讀效率。

未來研究將聚焦於資料範圍擴充、分析深度提升、Agent 能力完善與平台化落地，目標是逐步將目前的研究原型發展為可實際支援管理決策的資料分析系統。
