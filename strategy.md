# 專案策略文件

## 目標

本專案的核心目標，是建立一套在去識別化前提下仍可支援營收與庫存分析的資料處理與問答系統，並驗證其可行性。

## 策略主軸

### 1. 先穩定資料分析底座

- 建立固定的 Excel 匯入、欄位檢核與清洗流程
- 解析 mapping 並保留事業群、HQBU、平台之間的對應能力
- 產出可重複執行的趨勢、排名、異常與相關分析

### 2. 在既有管線上增量加入 AI 能力

- 不推翻既有分析流程
- 以 Tool Layer 封裝既有分析結果
- 讓 Main Agent 與各 Domain Agent 重用既有分析資料

### 3. 保持系統在 LLM 不穩定時仍可運作

- 問答邏輯保留 deterministic fallback
- 報告生成保留 fallback explanation
- 讓系統不因 Ollama 不可用而完全失效

### 4. 以 PoC 為先，逐步平台化

- 先完成可展示、可驗證的分析原型
- 再往測試、自動化、前端體驗與正式部署推進

## 目前策略重點

- 完成文件化
- 穩定 `uv` 環境
- 補齊 Git 協作基礎
- 明確區分目前完成與未來工作
- 將 2 人分工明確化

## 下一階段策略

- 強化 HQBU 與平台橋接邏輯
- 擴充更多資料欄位與資料源
- 完善 Multi-Agent 問答品質
- 建立正式測試與部署流程
